üîß VPS KVM Server Setup Guide

Step 1: Initial Server Setup & Security

1. SSH Access & Root Security

# Connect to your VPS via SSH
ssh root@your_vps_ip_address

# Update system packages to latest versions
apt update && apt upgrade -y

# Install essential security packages
apt install -y ufw fail2ban wget curl nano git

# Create a new sudo user (optional but recommended)
adduser yourusername
usermod -aG sudo yourusername

# Configure SSH key authentication
ssh-keygen -t rsa -b 4096
# Copy your public key to ~/.ssh/authorized_keys on the server

# Disable password-based SSH login
nano /etc/ssh/sshd_config
# Set: PasswordAuthentication no
systemctl restart ssh

2. Configure Firewall (UFW)

# Enable UFW firewall
ufw enable

# Allow SSH (important - do this before enabling firewall!)
ufw allow 22/tcp

# Allow HTTP and HTTPS
ufw allow 80/tcp
ufw allow 443/tcp

# Check firewall status
ufw status

# (Optional) Allow specific ports
ufw allow 3000/tcp    # Node.js apps
ufw allow 5432/tcp    # PostgreSQL
ufw allow 27017/tcp   # MongoDB

3. Setup Fail2Ban (Brute Force Protection)

# Install and start fail2ban
apt install -y fail2ban
systemctl start fail2ban
systemctl enable fail2ban

# Create local configuration
cp /etc/fail2ban/jail.conf /etc/fail2ban/jail.local

# Check fail2ban status
fail2ban-client status

Step 2: Install Web Server

Option A: Install Nginx

# Install Nginx
apt install -y nginx

# Start Nginx service
systemctl start nginx
systemctl enable nginx

# Check if Nginx is running
systemctl status nginx

# Test Nginx configuration
nginx -t

Option B: Install Apache (Alternative)

# Install Apache2
apt install -y apache2

# Enable required modules
a2enmod rewrite
a2enmod ssl
a2enmod proxy

# Start Apache service
systemctl start apache2
systemctl enable apache2

# Test Apache configuration
apache2ctl configtest

Step 3: Install Node.js & Applications

Install Node.js (LTS)

# Add NodeSource repository (Node.js 20.x LTS)
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -

# Install Node.js and npm
apt install -y nodejs

# Verify installation
node --version
npm --version

# Install PM2 (Process Manager for Node apps)
npm install -g pm2

# Check PM2 version
pm2 --version

Deploy Your Application

# Create application directory
mkdir -p /var/www/myapp
cd /var/www/myapp

# Clone your repository
git clone https://github.com/yourusername/your-app.git .

# Install dependencies
npm install

# Build your application (if needed)
npm run build

# Start with PM2
pm2 start npm --name "myapp" -- start

# Save PM2 configuration for system restart
pm2 save
pm2 startup

# View running processes
pm2 list
pm2 logs myapp

Step 4: Domain Configuration & SSL Certificate

Configure Nginx for Your Domain

# Create Nginx configuration
nano /etc/nginx/sites-available/myapp

# Add this content:
server {
    listen 80;
    listen [::]:80;
    server_name yourdomain.com www.yourdomain.com;
    
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

# Enable the site
ln -s /etc/nginx/sites-available/myapp /etc/nginx/sites-enabled/

# Test Nginx
nginx -t

# Reload Nginx
systemctl reload nginx

Install Free SSL Certificate (Let's Encrypt)

# Install Certbot
apt install -y certbot python3-certbot-nginx

# Generate SSL certificate (Let's Encrypt)
certbot certonly --nginx -d yourdomain.com -d www.yourdomain.com

# The certificate will be issued automatically and located at:
# /etc/letsencrypt/live/yourdomain.com/

# Update Nginx configuration with SSL
certbot install --nginx -d yourdomain.com -d www.yourdomain.com

# Test automatic renewal
certbot renew --dry-run

# Certificate will auto-renew 30 days before expiry

üíæ Database Installation & Setup

Install MongoDB

# Add MongoDB repository
curl -fsSL https://www.mongodb.org/static/pgp/server-7.0.asc | apt-key add -
echo "deb [ arch=amd64,arm64 ] https://repo.mongodb.org/apt/ubuntu jammy/mongodb-org/7.0 multiverse" | tee /etc/apt/sources.list.d/mongodb-org-7.0.list

# Install MongoDB
apt update
apt install -y mongodb-org

# Start MongoDB
systemctl start mongod
systemctl enable mongod

# Verify MongoDB is running
systemctl status mongod

# Connect to MongoDB shell
mongosh

# Create database and user
use myapp_db
db.createUser({
  user: "appuser",
  pwd: "strongpassword",
  roles: ["readWrite", "dbAdmin"]
})

Install PostgreSQL

# Install PostgreSQL
apt install -y postgresql postgresql-contrib

# Start PostgreSQL
systemctl start postgresql
systemctl enable postgresql

# Create database and user
sudo -u postgres psql

# In PostgreSQL shell:
CREATE DATABASE myapp_db;
CREATE USER myapp_user WITH PASSWORD 'strong_password';
ALTER ROLE myapp_user SET client_encoding TO 'utf8';
ALTER ROLE myapp_user SET default_transaction_isolation TO 'read committed';
GRANT ALL PRIVILEGES ON DATABASE myapp_db TO myapp_user;
\q

# Connect to database
psql -U myapp_user -d myapp_db

Install MySQL/MariaDB

# Install MySQL
apt install -y mysql-server

# Run security script
mysql_secure_installation

# Start MySQL
systemctl start mysql
systemctl enable mysql

# Create database and user
mysql -u root -p

# In MySQL shell:
CREATE DATABASE myapp_db;
CREATE USER 'myapp_user'@'localhost' IDENTIFIED BY 'strong_password';
GRANT ALL PRIVILEGES ON myapp_db.* TO 'myapp_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;

‚ö° VPS Optimization & Best Practices

üöÄ Performance Optimization

  • Enable Gzip compression in Nginx/Apache
  • Implement page caching with Redis/Memcached
  • Optimize database queries and indexes
  • Use CDN for static assets (Cloudflare, AWS CloudFront)
  • Enable HTTP/2 and HTTP/3 protocols
  • Compress images with WebP format

üîí Security Best Practices

  • Keep system and packages updated regularly
  • Use SSH keys instead of passwords
  • Enable SELinux or AppArmor
  • Regular backups (weekly minimum)
  • Monitor logs with tools like ELK Stack
  • Use environment variables for sensitive data
  • Implement DDoS protection (Cloudflare)

üìä Monitoring & Maintenance

  • Monitor CPU, RAM, disk usage with htop/top
  • Set up log rotation to manage disk space
  • Monitor application logs in real-time
  • Use tools like New Relic or Datadog for APM
  • Set up alerts for critical metrics
  • Regular backup verification

üéØ Cost Optimization

  • Choose appropriate plan size for your needs
  • Use referral codes for discounts
  • Upgrade only when necessary
  • Combine with CDN to reduce bandwidth costs
  • Use scheduled backups to external storage
  • Monitor and remove unused services
